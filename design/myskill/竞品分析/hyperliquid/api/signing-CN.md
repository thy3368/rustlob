# 签名

建议使用现有 SDK 而不是手动生成签名。签名可能出错的方式有很多。不正确的签名导致根据签名和有效负载恢复不同的签名者，并导致以下错误之一：

- `"L1 error: User or API Wallet 0x0123... does not exist."`
- `Must deposit before performing actions. User: 0x123...`

其中返回的地址与您签署钱包的公共地址不匹配。返回的地址也会因不同输入而变化。

不正确的签名不指示为什么不正确，这使得调试更具挑战性。要调试此问题，建议仔细阅读 Python SDK 并确保实现完全匹配。如果这不起作用，添加日志记录以找到输出分歧的位置。

## 常见错误：

1. 没有意识到有两种签名方案（Python SDK 方法是 `sign_l1_action` 与 `sign_user_signed_action`）。
2. 没有意识到字段顺序对 msgpack 很重要。
3. 数字尾随零的问题。
4. 地址字段中大写字符的问题。建议在签名和发送之前将小写任何地址。有时该字段被解析为字节，导致它在整个网络中自动小写。
5. 相信签名必须正确，因为在本地调用恢复签名者会产生正确的地址。恢复签名者的有效负载基于操作构建，不一定匹配。
