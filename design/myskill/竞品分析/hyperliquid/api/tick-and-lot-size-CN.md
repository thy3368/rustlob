# Tick 和 lot size

价格 (px) 和数量 (sz) 都有可接受的最大小数位数。

价格最多可以有 5 位有效数字，但不超过 `MAX_DECIMALS - szDecimals` 位小数，其中 `MAX_DECIMALS` 对于永续是 6，对于现货是 8。整数价格总是允许的，无论有效数字数量。例如 `123456` 是有效价格，即使 `12345.6` 不是。

数量四舍五入到该资产的 `szDecimals`。例如，如果 `szDecimals = 3`，则 `1.001` 是有效数量，但 `1.0001` 不是。

资产的 `szDecimals` 在 info 端点的 meta 响应中找到。

## 永续价格示例

- `1234.5` 有效但 `1234.56` 不是（太多有效数字）
- `0.001234` 有效，但 `0.0012345` 不是（超过 6 位小数）
- 如果 `szDecimals = 1`，`0.01234` 有效但 `0.012345` 不是（超过 `6 - szDecimals` 位小数）

## 现货价格示例

- 如果 `szDecimals` 是 0 或 1，`0.0001234` 有效，但如果 `szDecimals` 大于 2 则不是（超过 8-2 位小数）
