# 桥接

存入桥接的资金由验证者签名，当超过 2/3 的质押权力签名存款时入账。

从 Hyperliquid 提款立即从 L1 余额中扣除，验证者将提款作为单独交易签名。当 2/3 的质押权力签名提款时，可以向桥接发送 EVM 交易请求提款。

提款请求后，有一个争议期，在此期间如果恶意提款与 Hyperliquid 状态不匹配，可以锁定桥接。需要 2/3 质押加权验证者集的冷钱包签名才能解锁桥接。

争议期后，发送最终确定交易，将 USDC 分配到相应的目标地址。有类似机制来维护桥接合约上的活跃验证者集及其相应质押。

提款不需要用户的任何 Arbitrum ETH。相反，用户在 Hyperliquid 上支付 1 USDC 的提款 gas 费，以覆盖验证者的 Arbitrum gas 成本。

桥接及其与 L1 质押相关的逻辑已由 Zellic 审计。完整的桥接代码请参见 Hyperliquid Github 仓库，审计报告请参见审计部分。
