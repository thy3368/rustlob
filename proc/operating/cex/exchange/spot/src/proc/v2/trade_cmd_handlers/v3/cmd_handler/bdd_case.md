# BDD Case for match_order_handler.rs

设计目标：全面覆盖订单撮合场景，验证部分成交逻辑正确性

## 普通限价单 - 基础场景

### 场景1: 订单簿为空，无对手订单
- 挂一笔买单5，订单簿为空，检查无成交，订单状态变为OPEN
- 挂一笔卖单5，订单簿为空，检查无成交，订单状态变为OPEN

### 场景2: 完全成交 - 单一对手
- 订单簿已有卖单5，挂单5，买单5买入，检查成交数量为5，卖单被完全成交
- 订单簿已有买单5挂单5，卖单5卖出，检查成交数量为5，买单被完全成交

### 场景3: 部分成交 - taker数量大于单个maker
- 挂一笔卖单5，再挂一笔买单3，检查买单只成交3，卖单剩余2 → 验证部分成交后剩余数量正确
- 挂一笔买单5，再挂一笔卖单3，检查卖单只成交3，买单剩余2 → 验证部分成交后剩余数量正确

### 场景4: 部分成交 - taker数量大于多个maker总和
- 订单簿已有卖单3+卖单3，买单8买入，检查成交2笔共6，订单簿清空 → 验证多笔成交聚合
- 订单簿已有买单3+买单3，卖单8卖出，检查成交2笔共6，订单簿清空 → 验证多笔成交聚合

### 场景5: 部分成交 - taker数量大于部分maker
- 订单簿已有卖单3+卖单5，买单6买入，检查成交2笔（3+3），剩余挂单卖单5 → 验证按价格优先/数量优先成交
- 订单簿已有买单3+买单5，卖单6卖出，检查成交2笔（3+3），剩余挂单买单5 → 验证按价格优先/数量优先成交

### 场景6: 精确成交 - 数量完全匹配
- 挂一笔卖单5，再挂一笔买单5，检查两单完全成交，订单簿清空 → 验证精确成交无剩余
- 挂一笔买单5，再挂一笔卖单5，检查两单完全成交，订单簿清空 → 验证精确成交无剩余

## 边界场景

### 场景7: taker数量为零
- 挂买单0，检查无成交 → 验证零数量不产生交易

### 场景8: 单笔成交 - 数量差异大
- 订单簿已有卖单10，买单1买入，检查成交数量为1，卖单剩余9 → 验证小额成交正确
- 订单簿已有买单10，卖单1卖出，检查成交数量为1，买单剩余9 → 验证小额成交正确

### 场景9: 多笔小额成交聚合
- 订单簿已有卖单1+卖单1+卖单1+卖单1+卖单1，买单5买入，检查成交5笔，每笔1，订单簿清空 → 验证累积成交正确

## 价格优先场景

### 场景10: 价格匹配 - 买一价等于卖一价
- 订单簿已有卖单10@5000，买单5@5000，检查按5000成交 → 验证价格相同成交

### 场景11: 价格交叉（套利）
- 订单簿已有卖单10@5000，买单5@5100，检查按5000成交 → 验证taker接受更优价格

## 组合场景

### 场景12: 多笔部分成交 + 剩余挂单
- 订单簿已有卖单2+卖单2+卖单2，买单7买入，检查成交3笔共6，剩余1未成交 → 验证多 maker 部分成交

### 场景13: 成交ID唯一性
- 挂卖单1+卖单2+卖单3，买单5买入，检查每笔trade_id不同且递增 → 验证成交ID唯一

## 验证点总结

|-------|---------|

| 完全成交 | 2, 6 |
| 部分成交剩余taker | 3, 4, 5, 12 |
| 部分成交剩余maker | 3 |
| 多笔成交聚合 | 4, 5, 9, 12 |
| 边界值(0) | 7 |
| 边界值(1) | 8 |
| 价格优先 | 10, 11 |
| 唯一性 | 13 |
