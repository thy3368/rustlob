# 漏洞赏金计划

## 范围内

* 主网：导致节点或 API 服务器中断或逻辑错误的漏洞符合资格。
* 测试网：测试网独有的 HyperEVM 功能和 HyperCore 交互也包括在内，但非主网功能的赏金价值可能不同。
* 其他实验性测试网功能不在范围内，除非通过 Discord 或 Telegram 宣布。

## 提交流程

* 准备详细报告，包括复现步骤和概念证明，然后发送邮件至 bugbounty@hyperfoundation.org。
* 如果提交重复，第一个报告者获得认可。
* "将根据漏洞的严重性，为负责任的漏洞披露支付 USDC 奖励。"
* 对于遵守计划指南的善意研究，不会采取法律行动。
* 所有漏洞报告努力都受到重视。

## 禁止活动

* 针对主网代码进行测试；仅使用测试网或本地分叉。
* 网络钓鱼或社会工程。
* 大规模 DDoS 攻击（临时负载峰值可接受）。
* 第三方系统或应用测试。
* 勒索要求或威胁。
* 在修复和支付之前公开披露。
* 未经同意分享他人的个人或敏感信息。
* 通过漏洞获利超出计划奖励。
* 规避程序或未经授权的活动。

## 资格要求

* 仅提交至 bugbounty@hyperfoundation.org。
* 完成 KYC/KYB 要求。
* 需要能够在 Hyperliquid 上接收 USDC。
* 在获得授权披露之前保持机密。
* 发现必须可复现。
* 代码贡献者不能就其自己的贡献参与。

## 不合格情况

* 细节不足或缺少可复现示例。
* 需要不太可能用户行为的利用。
* 过时软件或不支持系统导致的问题。
* 需要 root 访问权限或设备修改的漏洞。
* 没有直接 Hyperliquid 影响的第三方库问题。
* 非安全漏洞，如轻微性能问题。
* 依赖于极端、不现实市场条件的漏洞。

## 一般条件

* 不符合资格的提交不获得支付。
* 有效性和分类判定完全由 Hyper Foundation 决定。
* "所有提交成为 Hyper Foundation 的财产。"

## 分类示例

* 严重 (<100 万 USDC)：重大用户资金损失或 L1 执行不变量违反。
* 高 (<5 万 USDC)：无错误状态的网络停机。
* 中等 (<1 万 USDC)：API 服务器性能问题。

严重性反映影响和可能性；支付在所述范围内变化。
