# 订单簿

HyperCore 状态包括每个资产的订单簿。订单簿的功能类似于中心化交易所。订单以 tick size 的整数倍价格下单，size 以 lot size 的整数倍计算。订单按价格-时间优先级匹配。

永续资产订单簿的操作引用清算所，因为所有仓位和保证金检查都在那里处理。保证金检查在开立新订单时进行，并在每次订单匹配时再次对挂单方进行。这确保了尽管挂单后预言机价格波动，保证金系统仍保持一致。

Hyperliquid L1 的一个独特方面是"内存池和共识逻辑在语义上 aware 与 HyperCore 订单簿交互的交易"。在一个区块内，操作按以下顺序排序：

1. 不向任何订单簿发送 GTC 或 IOC 订单的操作
2. 取消订单
3. 发送至少一个 GTC 或 IOC 的操作

在每个类别内，操作按区块提议者提议的顺序排序。修改订单根据它们放置的新订单进行分类。
