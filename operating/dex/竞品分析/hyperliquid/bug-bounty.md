# Bug Bounty Program

## In Scope

* Mainnet: bugs causing outages or logical errors on nodes or API servers qualify.
* Testnet: testnet-exclusive HyperEVM features and HyperCore interactions are included, though non-mainnet feature bounties may differ in value.
* Other experimental testnet features are out of scope unless announced via Discord or Telegram.

## Submission Process

* Prepare a detailed report with reproduction steps and proof of concept, then email bugbounty@hyperfoundation.org.
* First reporter receives credit if duplicates are submitted.
* "Rewards will be paid in USDC on Hyperliquid for responsible disclosure of bugs based on their severity."
* Legal action won't be pursued for good-faith research complying with program guidelines.
* All bug report efforts are valued.

## Prohibited Activity

* Testing against mainnet code; use testnet or local forks only.
* Phishing or social engineering.
* Large-scale DDoS attacks (temporary load spikes are acceptable).
* Third-party system or application testing.
* Ransom demands or threats.
* Public disclosure before fixes and payment.
* Sharing others' personal or sensitive information without consent.
* Profiting from vulnerabilities beyond program rewards.
* Circumventing procedures or unauthorized activities.

## Eligibility

* Submit exclusively to bugbounty@hyperfoundation.org.
* Complete KYC/KYB requirements.
* Ability to receive USDC on Hyperliquid required.
* Maintain confidentiality until authorized disclosure.
* Findings must be reproducible.
* Code contributors cannot participate regarding their own contributions.

## Ineligibility

* Insufficient detail or missing reproducible examples.
* Exploits requiring unlikely user behavior.
* Issues from outdated software or unsupported systems.
* Vulnerabilities needing root access or device modifications.
* Third-party library issues without direct Hyperliquid impact.
* Non-security bugs like minor performance problems.
* Bugs dependent on extreme, unrealistic market conditions.

## General Conditions

* Non-qualifying submissions receive no payment.
* Validity and classification determinations rest solely with Hyper Foundation.
* "All submissions become the property of the Hyper Foundation."

## Classification Examples

* Critical (<1M USDC): Significant user fund loss or L1 execution invariant violations.
* High (<50k USDC): Network downtime without incorrect state.
* Medium (<10k USDC): API server performance issues.

Severity reflects both impact and likelihood; payouts vary within stated ranges.
